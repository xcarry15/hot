# Hot2-S 架构优化执行计划

## 第一性原理

项目的最小业务闭环是：采集文章 → 处理正文 → AI 分析 → 人工审核 → 推送/公开。架构质量应优先保证：

1. 每种业务状态只有一个事实源。
2. 状态变更必须原子、可解释、可恢复。
3. Route 与组件只做适配，业务规则留在 Service。
4. 公开读模型不能被浏览统计等非内容写入污染。
5. 单进程与 SQLite 是当前明确边界，不引入队列、多实例和通用 Repository。
