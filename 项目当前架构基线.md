# 项目当前架构基线

> 更新时间：2026-07-13  
> 本文以当前 `main` 分支源码为准，是项目架构、入口和默认行为的唯一事实基线。README 负责快速开始，`CLAUDE.md` 负责协作约束，`bat/部署和更新方法.txt` 负责发布运维；三者不再重复维护业务规则。

## 1. 产品边界

Hot2 是单进程新闻聚合与飞书推送工具，主链路为：

```text
数据源采集 → 详情处理 → 去重/关键词过滤 → AI 分析 → 条件推送
```

当前页面仍有三个入口：文章流、抓取记录、设置。文章流面向最终用户；抓取记录和设置面向管理员。两类文章详情必须保持独立：

- `article-sheet.tsx`：文章流的用户详情，只展示用户阅读所需内容。
- `article-detail-sheet.tsx`：抓取记录的管理员后台详情，展示流水线状态、评分诊断、去重证据和维护操作。

## 2. 后台执行模型

- Next.js 单进程模块化单体；生产环境只运行一个 PM2 实例。
- `src/lib/execution.ts` 是唯一批量 Job 编排入口，内存互斥保证同一时间只有一个批量任务。
- Job 仍写入数据库，`Job` 的阶段、进度、错误数和 heartbeat 是任务状态事实源。
- 正常停止通过 `AbortSignal` 协作式取消；`/api/worker/stop` 负责触发停止。
- `src/lib/job-context.ts` 通过 `AsyncLocalStorage` 传递当前 Job ID，供进度写入使用；它不是队列。
- 调度器每分钟检查一次：自动抓取默认关闭；批量推送可按设置的 batch 模式和时间执行。
- `JobType.fastProcess` 仅为历史数据库记录保留的废弃兼容值；新代码不得创建该类型的新 Job。

### 流水线阶段

| 阶段 | 作用 | 入口 |
|---|---|---|
| `collect` | 采集数据源并写入/更新文章 | `/api/crawl`、单源重试 |
| `process` | 抓取详情、提取正文、执行过滤和去重 | `/api/crawl` |
| `ai` | 分析待处理文章，写入摘要、标签、评分和审计字段 | `/api/crawl` |
| `push` | 按统一推送条件投递未推送文章 | `/api/crawl` |
| `all` | 依次执行上述完整链路 | `/api/crawl` 默认阶段 |

管理员可分阶段执行，但分阶段入口属于高级运维能力；普通批量执行只应使用 `all`。

## 3. API 入口归属

- `POST /api/crawl`：唯一批量任务入口，支持 `all / collect / process / ai / push`；带 `sourceId` 时只采集该数据源。
- `POST /api/sources/retry`：只重试指定数据源的采集，不自动处理、AI 或推送全局文章。
- `POST /api/push`：只允许单篇文章推送；批量推送统一使用 `/api/crawl` 的 `push` 阶段。
- `POST /api/articles/refetch`：单篇重新抓取文章详情。
- `POST /api/articles/reprocess`：单篇重新执行 AI。
- `GET /api/crawl-log/status`：抓取记录页唯一任务快照接口，禁止从浏览器本地状态推断任务进度。
- `POST /api/worker/stop`：停止当前运行中的任务。
- `GET /api/cleanup` / `POST /api/cleanup`：维护统计与清理操作；没有独立清空推送日志入口，只有删除文章或全量破坏性清理时按数据完整性联动删除关联日志，日常只按保留策略清理。
- 设置、数据源、文章、推送日志、去重统计分别由对应 `/api/*` Route Handler 适配器调用 Service；Route 不承载事务和业务规则。

已移除的旧入口不得重新引入：`/api/jobs`、`/api/articles/refetch-batch`、独立队列/Job History API、独立批量候选推送 API。

## 4. 数据事实与投影

- `Job`：任务运行事实；不能用前端步骤、sessionStorage 或乐观状态替代。
- `FetchLog`：数据源采集日志。
- `PushLog`：按文章和 Webhook 目标记录成功/失败，是部分成功投递和重试幂等依据；不要直接清空。
- `DiscardedItem`：采集/处理阶段未入库条目的去重或过滤事实。
- `Article.aiStatus = skipped` 且存在 `dedupDetail`：AI 阶段去重事实。去重统计和清理必须同时读取两类事实。
- `DiscardedRetryAudit`：管理员重试未入库条目的不可变审计快照。
- `Article.rawContent / articleBody / cleanContent`：分别承担原始抓取缓存、提取后的正文缓存和清洗后的展示/分析内容，不能仅因字段相似而合并。
- `Source.totalArticles` 和 `Article.pushUrgency` 已不是数据库事实：来源文章数使用 Prisma `_count.articles`，推送紧急度由当前分数派生。
- 列表接口返回摘要投影和全库 facets；详情接口才返回正文、评分明细、去重证据和脱敏后的推送日志。

### 去重当前规则

规则唯一来源是 `src/contracts/dedup-settings.ts`，设置目录只引用它。当前默认值：

| 规则 | 默认值 |
|---|---:|
| 时间窗口 | 15 天 |
| 特定数字最少重叠数 | 2 |
| 正文 LCS 单段阈值 | 40 字符 |
| 正文 LCS 总长阈值 | 160 字符 |
| 品牌门控 | 开启 |
| 短文兜底阈值 | 1000 字符 |

标题 Jaccard 只作为证据/诊断，不单独触发去重；去重命中必须保留可解释的 `DedupEvidence`。

## 5. 设置与默认值

- `src/lib/settings-catalog.ts` 是设置 key、默认值、校验、敏感性、导出策略和前端默认值的唯一目录。
- `src/contracts/ai-provider.ts` 是 AI Provider 的 URL、默认模型和 Provider ID 唯一来源。
- `src/lib/preset-sources.ts` 是预设数据源唯一来源；`prisma/seed.ts` 只调用目录，不复制源定义。
- Webhook 默认值为空数组 `[]`，仓库不得保存真实 Webhook。
- `auto_crawl_enabled` 默认关闭；添加预设源默认禁用，手动启用本身不触发抓取。
- 敏感设置 GET 脱敏；明文只允许在受保护的 reveal/export 流程中按白名单读取。
- 修改设置目录、新增 Provider 或新增去重规则时，必须同步契约、校验、UI 和对应测试。

## 6. 数据库与发布基线

当前 migration 按序由 `prisma migrate deploy` 应用：

```text
20260101000000_init_baseline
20260102000000_drop_per_source_fields
20260103000000_add_job_progress_snapshot
20260104000000_add_log_query_indexes
20260711132028_b17_add_runtime_query_indexes
20260712100000_add_discarded_retry_audit
20260712150000_add_score_features
20260712170000_complete_p0_ai_audit
20260713100000_remove_redundant_article_caches
```

生产更新顺序：停止服务 → 备份 SQLite → `npm run db:migrate:deploy` → `npm run db:generate` → `npm run build` → 重启 PM2。日常更新不得使用 `db:reset`、`db:push` 或 `db:seed`；本地明确重建数据库时才使用破坏性命令。

日志保留由 `src/lib/log-retention.ts` 统一负责：FetchLog 30 天、PushLog 90 天但保留未完成全部投递的记录、已完成/失败 Job 30 天；不删除 Article、Source、DiscardedItem 和 pending/running Job。

## 7. 防漂移约定

1. 新增业务入口前先检查是否可归入现有 Route/Service；批量任务只能归入 `/api/crawl`。
2. 默认值只写入设置目录、Provider 契约或预设源目录之一；Seed 不复制业务默认值。
3. 数据库新增/删除字段必须同时提交 schema、migration、baseline 兼容处理和发布说明。
4. API 新增字段必须更新 DTO 投影、序列化函数和消费者；不得把 Prisma model 直接返回浏览器。
5. 页面状态只能使用对应服务端事实源；抓取记录使用 Job snapshot，文章详情组件不得跨页面合并。
6. 删除或废弃入口后，同时清理客户端调用、Service、测试、注释和部署文档。
7. 发布前至少执行 `npm run db:migrate:status`、`npm run db:generate` 和 `npm run build`；逻辑变更再执行对应测试。
