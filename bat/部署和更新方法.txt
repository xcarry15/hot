Hot2 部署和更新方法（精简版）
============================

生产目录：/www/wwwroot/hot.kfxz.cn
生产端口：3011
PM2 应用：h2-hot2（只能运行一个实例）
环境要求：Node.js >= 20.9.0、npm >= 10、PM2、sqlite3、rsync、unzip、curl


一、服务端首次初始化或完全重建
--------------------------------

此流程会先停止原有 PM2 进程，再清理旧项目代码，最后删除旧 SQLite、node_modules 和 .next，重新创建全新数据。
现有 .env 会保留；如果存在旧数据库，初始化脚本会先备份。

不要手工执行 rm -rf /www/wwwroot/hot.kfxz.cn/*：它可能误删配置，并且容易遗漏隐藏文件。项目资料统一由 rsync --delete 收敛清理，同时明确保留 .env、db/ 和 node_modules/；其中旧数据库和 node_modules 会在后续初始化脚本中备份或删除。

1. 本地双击 bat/run.bat，生成最新 zip。

2. 把 zip 上传到项目目录外：

  /www/wwwroot/h2-hot2-release.zip

不要直接把 zip 解压到 /www/wwwroot/hot.kfxz.cn，否则新版本已删除的旧文件可能残留。

3. 确认生产配置存在：

  ls -lh /www/wwwroot/hot.kfxz.cn/.env

如果是第一次部署，没有 .env，执行：

  mkdir -p /www/wwwroot/hot.kfxz.cn
  cd /www/wwwroot/hot.kfxz.cn
  echo 'DATABASE_URL=file:../db/custom.db' > .env
  echo "API_TOKEN=$(openssl rand -hex 32)" >> .env
  echo 'NEXT_PUBLIC_SITE_URL=https://hot.kfxz.cn' >> .env

4. 在宝塔终端完整复制下面这段命令。

括号中的内容运行在子 Shell；即使出错，也不会因为 set -e 断开当前宝塔终端。
Windows 生成的 zip 可能让 unzip 返回“使用反斜杠路径”的警告码 1，下方命令已正确兼容。

  (
  set -Eeuo pipefail

  APP=/www/wwwroot/hot.kfxz.cn
  ZIP=/www/wwwroot/h2-hot2-release.zip
  TMP=/tmp/h2-hot2-init

  test -s "$ZIP"
  test -s "$APP/.env"

  rm -rf "$TMP"
  mkdir -p "$TMP"

  set +e
  unzip -q "$ZIP" -d "$TMP"
  UNZIP_CODE=$?
  set -e

  if [ "$UNZIP_CODE" -gt 1 ]; then
    echo "错误：ZIP 解压失败，状态码 $UNZIP_CODE"
    exit 1
  fi

  LOCK_FILE="$(find "$TMP" -maxdepth 4 -type f -name package-lock.json -print -quit)"
  if [ -z "$LOCK_FILE" ]; then
    echo "错误：ZIP 中没有 package-lock.json"
    exit 1
  fi

  RELEASE_ROOT="$(dirname "$LOCK_FILE")"
  test -f "$RELEASE_ROOT/package.json"
  test -f "$RELEASE_ROOT/scripts/init-production.sh"

  echo "识别到项目目录：$RELEASE_ROOT"

  # 发布包验证成功后再停止旧进程，避免无效 zip 导致站点提前离线
  pm2 stop h2-hot2 >/dev/null 2>&1 || true
  pm2 delete h2-hot2 >/dev/null 2>&1 || true

  rsync -a --delete \
    --exclude='.env' \
    --exclude='.env.*' \
    --exclude='db/' \
    --exclude='node_modules/' \
    "$RELEASE_ROOT/" "$APP/"

  cd "$APP"
  ls -lh package.json package-lock.json scripts/init-production.sh

  rm -rf "$TMP"
  echo "项目文件同步完成"
  )

  echo "子 Shell 退出码：$?"

成功结果应显示：

  项目文件同步完成
  子 Shell 退出码：0

5. 执行一键初始化：

  cd /www/wwwroot/hot.kfxz.cn
  bash scripts/init-production.sh

输入 RESET 确认。脚本会自动完成：

- 再次确认旧 PM2 应用已停止（可重复执行）
- 备份并删除旧 SQLite
- 全新安装依赖
- 应用 migration、生成 Prisma Client、写入种子数据
- 优化数据库、生产构建、启动 PM2
- 检查本机和正式域名 /api/health

完整顺序是：验证 zip -> 停止 PM2 -> rsync 清理旧代码并同步新代码 -> 备份旧数据库 -> 全新初始化 -> 启动 PM2。

首次配置 PM2 开机启动时，再执行：

  pm2 startup

复制执行它输出的命令，然后执行：

  pm2 save
  rm -f /www/wwwroot/h2-hot2-release.zip


二、日常更新（已经自动化）
--------------------------

正常情况下不需要登录宝塔，也不需要手工上传 zip。

  推送或合并到 master
    -> GitHub Actions 自动测试
    -> 测试成功
    -> 自动备份数据库并部署生产环境

查看运行状态：

  https://github.com/xcarry15/hot/actions

手动运行 Deploy production 会绕过 CI，只用于重新部署已经验证过的提交。

自动部署会保留服务器上的：

- .env
- db/custom.db
- node_modules（同步阶段保留，随后 npm ci 更新）

自动部署失败不会自动恢复旧代码或回滚 migration。需要恢复时，应使用部署前生成的数据库备份，并重新部署一个明确可用的提交。


三、本地一键初始化
------------------

Windows 双击：

  bat/本地一键初始化.bat

该 bat 只负责启动纯英文 PowerShell 脚本 bat/local-init.ps1，避免 Windows CMD 因中文编码或复杂命令解析而闪退。初始化失败时窗口会保留并显示错误信息。

确认后会自动：

- 停止当前项目的本地 Node 进程
- 删除本地 SQLite、node_modules 和 .next
- 保留现有 .env；没有时从 .env.example 创建
- npm ci、migration、Prisma generate、seed、数据库优化
- 启动开发服务并打开 http://localhost:3011

该流程不保留本地历史数据，也不会自动运行测试或生产构建。


四、只清理业务历史，保留配置
----------------------------

如果项目已经正常运行，只想重新采集文章，不需要重建整个服务器。

进入后台：

  设置 -> 数据 -> 一键清空所有数据

它会删除文章、事件、任务、采集和推送历史，但保留数据源、关键词、系统设置和推送目标。


五、常用故障检查
----------------

  pm2 status
  pm2 logs h2-hot2 --lines 100
  curl --fail --silent --show-error http://127.0.0.1:3011/api/health
  curl --fail --silent --show-error https://hot.kfxz.cn/api/health

检查数据库 migration：

  cd /www/wwwroot/hot.kfxz.cn
  npm run db:migrate:status

重启应用：

  pm2 restart h2-hot2 --update-env

修改 Nginx 配置后才需要：

  nginx -t && nginx -s reload

普通项目更新不要清理服务器全局 Nginx 缓存，也不需要 reload Nginx。


六、备份和磁盘清理
------------------

数据库备份目录：

  /www/backup/h2-hot2

查看占用：

  du -sh /www/backup/h2-hot2 ~/.pm2/logs

查看超过 30 天的备份：

  find /www/backup/h2-hot2 -mindepth 1 -maxdepth 1 -type d -mtime +30 -print

确认不再需要后删除：

  find /www/backup/h2-hot2 -mindepth 1 -maxdepth 1 -type d -mtime +30 -exec rm -rf -- {} +


七、安全提醒
------------

- 不要把 .env、API_TOKEN、SSH 私钥、Webhook、数据库或部署 zip 提交到 GitHub。
- 不要在聊天、文档或命令日志中回显密码、API_TOKEN 或 SSH 私钥。
- 生产环境不要执行 prisma db push、db:danger:reset 或 db:seed，除非明确进行本文件第一节的全新初始化。
- PM2 禁止使用 -i max 或 cluster 模式，本项目只能运行一个实例。
