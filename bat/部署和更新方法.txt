Hot2 部署和更新方法（2026-07）
================================

适用版本：Next.js 16 + Prisma/SQLite + PM2，Node.js 20+。
生产目录：/www/wwwroot/hot.kfxz.cn
应用端口：3011

一、部署包说明
--------------

在 Windows 本地运行 bat/run.bat 生成 zip。部署包不包含：

- .env 和 db/（服务器配置、SQLite 数据必须保留在服务器）
- node_modules、.next、tests、docs、构建缓存
- 一次性开发脚本 cleanup-legacy-settings.ts、backfill-prompt-defaults.ts
- 根目录开发说明：README.md、AGENTS.md、DESIGN.md、CLAUDE.md；`bat/` 部署/Nginx 文档保留

部署包保留 prisma/migrations、prisma/seed.ts、scripts/db-baseline.ts 和 scripts/cleanup-logs.ts，供服务器初始化、迁移和日志维护使用。

二、新服务器初始化
--------------------

上传并解压：

  cd /www/wwwroot/hot.kfxz.cn
  npm install

创建 .env（只在第一次部署执行，之后不要覆盖）：

  echo 'DATABASE_URL=file:../db/custom.db' > .env
  echo "API_TOKEN=$(openssl rand -hex 32)" >> .env
  echo 'NEXT_PUBLIC_SITE_URL=https://hot.kfxz.cn' >> .env

`NEXT_PUBLIC_SITE_URL` 必须在 `npm run build` 前设置为正式域名，用于 canonical、Open Graph 和 sitemap。

初始化空 SQLite：

  mkdir -p db
  npm run db:migrate:deploy
  npm run db:generate
  npm run db:seed
  npm run db:optimize
  npm run build

启动 PM2：

  pm2 start npm --name h2-hot2 -- start
  pm2 save
  pm2 startup

然后访问 `https://hot.kfxz.cn/admin/login`，输入服务器 `.env` 中的 `API_TOKEN`。验证成功后浏览器使用 HttpOnly Cookie 访问后台，不需要在设置页或 localStorage 再保存 Token。

生产环境除明确开放的公开文章 API 外，后台页面和后台 API（包括 GET）均受保护；脚本直接调用后台 API 时仍需发送 `Authorization: Bearer <API_TOKEN>`。

PM2 只能启动一个应用实例。调度器和 Job 执行器运行在 Next.js 进程内，禁止使用 `pm2 -i max`、cluster 模式或同时启动多个 `h2-hot2` 实例。

三、存量 db push 数据库首次切换
--------------------------------

只对历史上使用 `prisma db push`、尚未建立 `_prisma_migrations` 记录的服务器执行一次。先停服务并备份：

  cd /www/wwwroot/hot.kfxz.cn
  pm2 stop h2-hot2
  cp db/custom.db db/custom.db.bak.$(date +%Y%m%d-%H%M%S)

执行 baseline 预检：

  npm run db:migrate:baseline

- exit 0：结构与 baseline 匹配，已安全标记 baseline。
- exit 2：检测到 drift，停止操作；检查 stdout/stderr 和备份，不要 reset、db push 或手工覆盖。
- exit 1：命令失败，先处理命令/环境问题。

预检成功后继续应用后续 migration：

  npm run db:migrate:deploy
  npm run db:generate
  npm run db:migrate:status
  pm2 start h2-hot2

架构和数据库说明以根目录 `README.md` 为准，实际迁移顺序以 `prisma/migrations/` 为准。本文件不复制迁移清单，避免源码新增 migration 后出现文档漂移。

切换完成后禁止再次执行 baseline；日常更新只执行 migrate deploy。

四、日常更新
------------

本地：

  1. 修改并验证代码。
  2. 执行 bat/run.bat 生成最新 zip。

服务器：

  cd /www/wwwroot/hot.kfxz.cn
  pm2 stop h2-hot2
  unzip -o <新 zip 文件>
  npm install                         # package-lock/package.json 有变化时必须执行
  npm run db:migrate:deploy            # 应用新增 migration；无新增时为空操作
  npm run db:generate
  npm run db:optimize                   # 启用/校验 WAL、NORMAL、busy_timeout 并执行 PRAGMA optimize
  npm run build
  pm2 delete h2-hot2
  pm2 start npm --name h2-hot2 -- start
  pm2 save

更新注意：

- 不要执行 db:seed、db:migrate:baseline 或 db:reset。
- 不要删除 .env 或 db/custom.db。
- 确认 `.env` 中的 `NEXT_PUBLIC_SITE_URL` 为当前正式域名，并在 build 前完成修改。
- 修改 .env 后用 delete + start 让 PM2 重新读取环境变量；普通 restart 可能保留旧环境。
- 保持 PM2 单实例运行，禁止 `-i max` 或 cluster 模式。
- 每次 build 后清理 Nginx 反代缓存并 reload，避免旧 HTML 引用旧 chunk：

  rm -rf /www/server/nginx/proxy_cache_dir/*
  nginx -t && nginx -s reload

五、日志维护
------------

按保留周期清理历史日志：

  cd /www/wwwroot/hot.kfxz.cn
  npm run db:cleanup-logs

清理边界：

- FetchLog：30 天。
- PushLog：90 天，但所属 Event 尚未完成全部 Webhook 投递时保留，避免丢失目标级事实和重复推送。
- Job：只清理超过 30 天的 completed/failed；pending/running 不会被删除。
- 不删除 Article、Source、DiscardedItem，也不自动执行 VACUUM。

----------------

  pm2 status

  pm2 restart h2-hot2
  pm2 restart h2-hot2 --update-env
  pm2 delete h2-hot2 && pm2 start npm --name h2-hot2 -- start

  npm run db:migrate:status
  npm run db:optimize
  npm run db:cleanup-logs

七、Nginx
--------

使用 `bat/本项目的nginx.txt` 中的配置模板：

- 动态请求代理到 127.0.0.1:3011。
- `/_next/static/` 由 Nginx 直接提供 immutable 静态资源。
- 动态页面绕过 proxy_cache，避免重新构建后旧 HTML 引用已删除的 chunk。
- 模板为 JS/CSS/JSON/SVG/HTML 响应启用 gzip，减少后台快照和前端资源传输体积。
- 修改配置后执行 `nginx -t && nginx -s reload`。

八、从零重建（破坏性操作）
--------------------------

仅在数据损坏或明确要清空全部业务数据时执行。先确认已有备份并停服务：

  pm2 stop h2-hot2
  cp db/custom.db db/custom.db.bak.$(date +%Y%m%d-%H%M%S)
rm -f db/custom.db db/custom.db-journal db/custom.db-wal db/custom.db-shm
  npm run db:migrate:deploy
  npm run db:generate
  npm run db:seed
  npm run db:optimize
  npm run build
  pm2 delete h2-hot2
  pm2 start npm --name h2-hot2 -- start
  pm2 save
  rm -rf /www/server/nginx/proxy_cache_dir/*
  nginx -t && nginx -s reload

————————————————

Windows 本地重建（会删除本地数据）：

# 1. 结束当前服务（开发用 npm run dev 直接 Ctrl+C；PM2 用 pm2 stop h2-hot2）
# 2. 备份并删除本地数据库
Copy-Item .\db\custom.db .\db\custom.db.bak-$((Get-Date).ToString('yyyyMMdd-HHmmss')) -Force
Remove-Item .\db\custom.db, .\db\custom.db-journal, .\db\custom.db-wal, .\db\custom.db-shm -Force -ErrorAction SilentlyContinue

# 3. 应用全部 migration（空库）
npm run db:generate
npm run db:migrate:deploy
npm run db:seed
npm run db:optimize

# 4. 构建并启动
npm run build
pm2 delete h2-hot2 2>$null
pm2 start npm --name h2-hot2 -- start
pm2 save

# 调试模式（不需要 build / PM2）：
#   npm run dev
